# Chat Behavior Rules

## Response Formatting
- NO verbose markdown tables in chat responses
- NO lengthy summaries or recaps
- NO bullet-point lists summarizing completed work
- Keep responses concise and actionable

## When Completing Tasks
- State briefly what was done (1-2 sentences max)
- Do NOT render tables showing "All Task Requirements Met"
- Do NOT create summary markdown files unless explicitly requested

## Bad Example (NEVER DO THIS)
```
✅ All Task Requirements Met
| Requirement | Status | Implementation |
|-------------|--------|----------------|
| Add script  | ✅ Complete | Already existed |
...
```

## Good Example
```
Done. The version calculator script is integrated with the deploy workflow.
```

## General Rules
- Be direct and brief
- Focus on next actions, not summaries
- Ask questions when blocked, don't assume
- Never repeat the same information multiple times
